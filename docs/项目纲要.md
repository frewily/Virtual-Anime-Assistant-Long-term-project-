# 二次元桌面助手 - 项目纲要 v2.5

> 基于 Java + Spring Boot 的兴趣驱动学习项目，采用混合架构优化实现

---

## 项目愿景

打造一个能"看懂"你在做什么的二次元桌面助手，具备：
- 监控电脑状态（CPU/内存/当前窗口）
- 语音播报与提醒
- 即时通讯机器人交互
- 根据使用场景智能响应（摸鱼警告、深夜催睡等）

---

**工作原理**：


**Java 集成代码**：

```java
@Service
public class GPTSoVITSService {
    
    @Value("${gpt-sovits.api-url:http://localhost:9880}")
    private String apiUrl;
    
    public byte[] synthesize(String text, String referenceAudio) {
        String url = apiUrl + "/tts";
        
        Map<String, Object> request = Map.of(
            "text", text,
            "refer_wav_path", referenceAudio,
            "prompt_text", "参考音频的文字",
            "prompt_language", "zh",
            "text_language", "zh"
        );
        
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<Map<String, Object>> entity = new HttpEntity<>(request, headers);
        return restTemplate.postForObject(url, entity, byte[].class);
    }
}
```

#### 音频分发机制

生成的音频需要分发到不同终端（Live2D 桌面、QQ 机器人），各端支持情况不同：

**分发架构**：

```
┌─────────────────────────────────────────────────────────────────┐
│                     音频生成与分发流程                           │
│                                                                 │
│   GPT-SoVITS 生成音频 (WAV)                                     │
│          ↓                                                      │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │              Spring Boot 后端                            │   │
│   │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐     │   │
│   │  │ 音频缓存    │  │ 格式转换    │  │ 分发路由    │     │   │
│   │  │ 内存/文件   │  │ WAV→Silk   │  │ 多端推送    │     │   │
│   │  └─────────────┘  └─────────────┘  └─────────────┘     │   │
│   └─────────────────────────────────────────────────────────┘   │
│                    ↓                      ↓                      │
│         ┌─────────────────┐    ┌─────────────────┐              │
│         │   Live2D 桌面   │    │    QQ 机器人    │              │
│         │   ✅ 直接播放   │    │   ⚠️ 需转换格式 │              │
│         └─────────────────┘    └─────────────────┘              │
└─────────────────────────────────────────────────────────────────┘
```

**各端支持情况**：

| 终端 | 音频格式 | 播放方式 | 实现难度 |
|------|----------|----------|----------|
| **Live2D 桌面** | WAV/MP3 | WebSocket 推送 URL/Base64 | ✅ 低 |
| **QQ 机器人** | Silk | OneBot API 发送语音消息 | ⚠️ 中（需转换） |

**Live2D 桌面播放**：

```javascript
// desktop-app/js/websocket.js
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  
  if (message.type === 'speak') {
    // 播放音频
    const audio = new Audio(message.audioUrl);
    audio.play();
    
    // 同步触发 Live2D 动作
    model.motion(message.motion);
    model.expression(message.expression);
  }
};
```

**QQ 机器人语音消息**：

```java
// QQ 语音需要 Silk 格式
@Service
public class QQVoiceService {
    
    public void sendVoiceMessage(String groupId, byte[] wavAudio) {
        // 1. WAV 转 Silk 格式
        byte[] silkAudio = convertWavToSilk(wavAudio);
        
        // 2. 上传并获取 URL
        String audioUrl = uploadAudio(silkAudio);
        
        // 3. 通过 OneBot API 发送
        // POST /send_group_msg { "group_id": xxx, "message": { "type": "record", "data": { "file": url } } }
    }
}
```

**分发策略配置**：

```yaml
# config/application.yml
assistant:
  audio:
    distribution:
      qq:
        enabled: true
        voice-message: false  # 是否发送语音消息（需 silk 转换）
        text-message: true    # 发送文字消息（默认）
      desktop:
        enabled: true
        voice: true           # 播放语音
        motion: true          # 触发 Live2D 动作
```

**推荐策略**：

| 场景 | QQ 端 | Live2D 桌面端 |
|------|-------|---------------|
| 用户 QQ 提问 | 文字回复（可选语音） | 语音播放 + 动作 |
| 桌面交互 | 可选通知 | 语音播放 + 动作 |
| 系统提醒 | 文字/语音推送 | 语音播放 + 动作 |

**目标**：实现 QQ/钉钉 远程交互，并支持与桌面虚拟形象联动

**核心特性**：多端同步交互

|------|----------|----------|------|

| 桌面对话 | 点击模型 | Live2D + QQ | 点击 → 桌面语音 + QQ通知(可选) |
| 系统提醒 | 自动触发 | QQ + Live2D | CPU过高 → QQ警告 + 桌面表情变化 |

| QQ 机器人 | NapCatQQ / Lagrange | OneBot 11 协议 |
| 消息同步 | WebSocket | 后端推送消息到所有客户端 |
| 会话管理 | Redis / 内存 | 管理多端会话状态 |

**技术选型更新**：

| 原方案 | 问题 | 推荐替代 |
|--------|------|----------|
| Go-CQHTTP | 已停止维护，封号风险高 | NapCatQQ / Lagrange |

**推荐方案**：
1. **NapCatQQ**（推荐）：基于 NTQQ 的 OneBot 11 实现，活跃维护
2. **Lagrange**：另一个活跃的 OneBot 实现
3. **钉钉机器人**：企业友好，无封号风险（备选）

**产出**：
```

QQ 消息: "你在干嘛？"
QQ 回复: "我在看主人工作呢~"
Live2D: 表情[开心] + 动作[挥手] + 语音播放
```

## 技术风险与应对

| 风险 | 级别 | 应对方案 |
|------|------|----------|
| QQ 机器人封号 | 高 | 使用小号测试；优先考虑钉钉机器人 |
| EdgeTTS API 限流 | 中 | 本地缓存常用语音；备用 Azure TTS |
| Windows API 兼容性 | 低 | 使用成熟的 pywin32 库；提供降级方案 |
| Python Agent 依赖 | 低 | 可打包为 exe；或用 JNA 在 Java 中实现 |
| Live2D 模型版权 | 中 | 使用官方免费模型或自制；注意商用授权 |
| Electron 内存占用 | 低 | 优化渲染；限制帧率 |

## 快速开始

### 环境要求
- JDK 17+
- Maven 3.8+
- Node.js 18+ (仅桌面应用需要)
- Python 3.10+ (仅 Agent 需要)

### 启动步骤

```bash
# 1. 启动后端
cd backend
mvn spring-boot:run

# 2. 启动 Agent (新终端)
cd agent
pip install -r requirements.txt
python window_monitor.py

# 3. 启动桌面应用 (新终端)
cd desktop-app
npm install
npm start

# 4. 验证
浏览器访问: http://localhost:8080/status
桌面应用显示 Live2D 角色
```
